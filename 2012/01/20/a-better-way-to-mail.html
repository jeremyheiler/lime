A Better Way to Mail

Posted on January 20, 2012

At the time of this writing, Clojure doesn't have any built-in mail libraries. Also, all of the third party mail libraries that I have seen depend on JavaMail. This is fine for anyone who wants to simply send mail using the JVM. The problem is that the JavaMail API was built for Java, which leaves much to be desired in terms of expressiveness and extensibility in a Clojure environment.

I believe there is an opportunity for a better and simpler solution to send mail with Clojure. Sending mail whould be as simple as a function call, but it should also be simple to dig into the protocols and use them to their fullest extent. There also is more than just the JVM to consider, and I believe having a common mail library that can easily be ported to other Clojure platforms would be incredibly useful.

In order to kick start this endeavor, I have begun implementing an SMTP client called Lime. The short-term goal for this project is to build a low-level API for managing the client-side of an SMTP session. The idea is that you should be able to interact with an SMTP server inside a REPL as if you were using telent to do the same. The next step would then be to run pre-defined mail sessions, and everything else that goes into building a high-level API.

This will take time to implement, but the effort will be worth it. I believe the Clojure community will be better off having its own mail libraries, rather than depend on platform support. If you wish to help out, hit up the Lime project page and check back here for progress updates. I plan on attending Clojure/West so maybe I will see you there!
